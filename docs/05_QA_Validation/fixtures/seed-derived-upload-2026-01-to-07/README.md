# 기존 SemuAgent 샘플 기반 업로드 검증 자료

이 폴더의 Excel 파일은 별도의 가상 거래를 만든 자료가 아닙니다. SemuAgent 첫 실행 샘플의 정본인 `reconciliation-sample-generator.ts`와 `buildSamplePayrollLines`에서 거래일·상대방·금액·매칭 상태를 직접 읽어, 각 원천 기관이 주는 형태로 다시 렌더링한 검증 자료입니다.

## 범위

| 원천 | 파일 | 정본 행 |
| --- | --- | ---: |
| 은행 | 계좌별 거래내역 | 245건 |
| 법인카드 | 카드사별 이용내역 + 부가세 처리 1건 | 106건 |
| 홈택스 | 전자세금계산서 매출·매입 목록 | 354건 |
| 급여시스템 | 2026-06 급여대장 | 11명 |

은행·카드 350건 중 336건은 같은 거래일·금액의 세금계산서와 연결됩니다. 나머지는 기존 샘플 그대로 은행 미분류 7건, 개인사용 의심 카드 7건, 미입금/미수금 세금계산서 14건입니다. 부가세 처리용 기존 확정 행 5건도 포함했습니다.

## 형식 원칙

- 은행: 거래일자·적요·입금액·출금액·거래후잔액 형태
- 카드사: 승인일자·가맹점명·이용금액·승인번호 형태
- 홈택스: 작성일자·거래구분·상호·공급가액·세액·합계금액 형태
- 급여: 실제 시드 직원별 귀속연월·기본급·식대·수당·고용형태 형태

거래 금액과 상대방·일자는 정본 값을 바꾸지 않았습니다. 은행 거래후잔액, 승인시간·번호처럼 정본에 없는 표시용 필드는 원천 파일 모양을 만들기 위한 값이며, 대조·분류의 정본 값이 아닙니다. 일반 세금계산서의 공급가액·세액은 기존 정본의 합계금액을 보존하면서 10% 과세로 역산한 표시값입니다. 기존 시드가 별도로 확정한 영세율·면세·공통매입·공제 가능 행은 그 확정 값을 사용합니다.

## 재생성

```bash
pnpm exec tsx scripts/qa/create-seed-derived-upload-fixtures.ts
```

기본 출력은 이 폴더입니다. 다른 안전한 임시 폴더로 만들려면 경로를 첫 번째 인자로 넘깁니다.

PG·현금영수증 원천 자료는 기존 첫 실행 샘플 정본에 없으므로, 이 자료에 임의로 추가하지 않았습니다.

## CUI-4 Closeout 사용

이 파일 묶음은 CUI-4 결과 카드와 자료수집 회귀의 입력 정본이다. 임의 합성 파일을 새로 만들지 않고 이 묶음을 사용해 파일 형식별 추출·결과 카드·같은 `period + sessionId` CTA·다른 세션 미혼입을 검증한다.

화면 확인은 localhost에서 할 수 있지만, 실제 Blob 업로드 callback과 DB 결과 행 생성은 공개 URL이 필요하다. 따라서 업로드 E2E는 production 데이터와 분리된 Preview/staging 환경에서 실행하고, 종료 후 QA 행과 Blob을 정리한다.
