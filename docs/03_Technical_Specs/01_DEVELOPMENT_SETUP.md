## Reuse Guard

Before reusing a JARYO-GIWA module:

1. Identify whether it is product-neutral, accounting-firm-specific, or company
   self-use-specific.
2. Keep product-neutral logic where possible.
3. Rename or adapt accounting-firm-specific entities before exposing them in
   UI/API.
4. Preserve tests when the business rule is unchanged.
5. Add new tests when the responsibility boundary changes.

## Local Setup

```bash
npm install
cp .env.local.example .env.local
npm run dev
```

Verification:

```bash
npm run lint
npm test
npm run build
```

`package-lock.json` should be generated by npm. If lockfile generation requires
network access, run `npm install --package-lock-only --ignore-scripts` in an
approved network environment.
