## 1. Purpose

이 문서는 과거 fcrypt 암호화 경로를 검토할 때 조사한 적합성 검정 제도를
보존하는 역사 기록이다. [NTS Crypto Spec §4.2](./31_JC030_NTS_CRYPTO_SPEC_ACQUISITION.md)의 조사에서
"국세청은 적합성 검정을 받은 세무프로그램(약 15개 업체)이 만든 파일만 파일변환신고로 받는다"는 정황이
확인됐다. 현재 Path 1은 공식 비암호화 업로드 양식만 지원하므로 이 조사는 활성 구현 게이트가 아니다.

**중요:** 이건 파일 규격(레이아웃)이나 암호화 알고리즘과 무관한 별도 게이트다. 파일을 완벽하게 만들어도
SemuAgent가 이 인증을 받지 못하면 파일변환신고 경로 자체가 막힐 수 있다.

## 3. Historical Open Questions (활성 백로그 아님)

아래 항목은 암호화 파일 제품 범위를 다시 승인할 때만 재검토한다. 현재 Path 1을
진행하기 위해 답을 구하거나 문의할 필요가 없다.

- [ ] 신청 자격: 사업자 규모(매출·업력) 제한이 있는가? 신생/소규모 서비스도 신청 가능한가?
- [ ] 신청 절차: 어디에(부서/시스템) 어떤 서류를 제출하는가?
- [ ] 심사 기간: 통상 몇 주/몇 개월이 걸리는가?
- [ ] 비용: 신청·심사·연회 비용이 있는가?
- [ ] 세목별 별도 검정인가, 한 번 통과하면 전 세목에 적용되는가?
- [ ] 검정 없이 생성한 파일을 사용자가 수동으로 업로드하면 어떻게 되는가(거부/경고/그냥 통과)?
- [ ] 검정 대기 중에도 "내용검증"까지는 통과 가능한가(제출만 막히는지, 검증 자체가 막히는지)?
