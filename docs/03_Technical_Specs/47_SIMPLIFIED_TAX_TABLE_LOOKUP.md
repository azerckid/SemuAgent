# 근로소득 간이세액표 직접 조회 엔진 (정규직 원천세)

> Created: 2026-07-12
> Layer: 03_Technical_Specs
> Related: [12_FIRST_RUN_SAMPLE_DATA_PRE_CODE_BRIEF](12_FIRST_RUN_SAMPLE_DATA_PRE_CODE_BRIEF.md), [00_BACKLOG](../04_Logic_Progress/00_BACKLOG.md) JC-012, [10_FIRST_RUN_SAMPLE_DATA_TEST_SCENARIOS](../05_QA_Validation/10_FIRST_RUN_SAMPLE_DATA_TEST_SCENARIOS.md)

## 1. 배경 / 문제

정규직 샘플의 소득세가 근거 없는 하드코딩 값이었다("간이세액표 계산이 아닌 것"). 사용자는
모든 정규직 소득세가 간이세액표에 근거하고, 공제대상가족수를 바꾸면 값이 움직이며,
직원명부-급여-원천세 세 화면이 부정합 없이 일치할 것을 요구했다.

## 2. 결정 — 파생 산식이 아니라 "직접 조회(lookup)"

`lib/payroll/simplified-tax-table.ts`는 별표2 제1호의 파생 산식(근로소득공제→과세표준→
기본세율→근로소득세액공제)을 재구현하지 **않는다.** 대신 **별표2 제6호 조견표의 실제 공표
세액**을 `월급여액 구간 × 공제대상가족수`로 조회한다.

- 근거: 소득세법 시행령 [별표2] 근로소득 간이세액표(개정 2024. 2. 29., 제189조제1항 관련).
  2026.2.27 개정은 8~20세 자녀세액공제 금액만 조정했고 자녀공제 전 기본 간이세액은 동일하다.
- 장점: 조회값이 법령 원문과 1:1로 대조 검증된다(추정·근사 없음). 잘못될 위험이 큰 근사식
  상수(연금보험료공제 가정 등)를 코드에 넣지 않는다.
- 자녀세액공제(별표2 제3호): 1명 12,500 / 2명 29,160 / 3명 이상 29,160 + 2명 초과 1명당 25,000.
  공제 후 음수는 0. 공제대상가족수 11명 초과(제4호)도 지원. 원천징수 비율 80/100/120% 지원.
- 지방소득세 = 소득세의 10%(원단위 절사).

## 3. 범위 / 비목표

- **범위 한정 lookup.** 현재 임베드한 급여 구간(샘플 정규직 6명이 안착하는 240·280·300·360·
  440·600만원 등)만 조회 가능하다. 구간 밖은 `{ inRange: false }`로 반환하고 **임의 추정하지
  않는다.** 시드 생성 시 범위 밖이면 즉시 실패시켜 잘못된 값이 저장되지 않게 한다.
- 이는 JC-027이 금지한 "신규 세액 계산 엔진(10%/11 근사 재계산)"과 다르다 — 공표 세액표를
  그대로 조회하는 것이며 세액을 자체 산출·추정하지 않는다.
- 프리랜서(사업소득 3.3%)·일용직((일급−150,000)×2.7%·소액부징수)에는 적용하지 않는다.
  원천세 팝오버는 고용형태별로 산출 근거 문구를 분리한다.

## 4. 데이터 / 스키마

- `payroll_employee_line.dependent_count`(마이그레이션 `0072`, NOT NULL DEFAULT 1): 본인 포함
  공제대상가족수. 정규직 소득세 조회 기준이며 프리랜서·일용직은 기본 1.
- 샘플 정규직 공제대상가족수: 김대표 4 / 이수민 3 / 박지훈 2 / 최민준 1 / 정하늘 2 / 오세린 1
  (모두 자녀 0). 값이 커질수록 같은 급여라도 소득세가 감소하는 표 효과를 보여준다.

## 5. 정합성 교차검증

`lib/payroll-workspace/consistency.ts`는 고용형태를 단일 기준으로 급여 라인 4대보험 부과가
기대 범위와 어긋나는지 판정한다(정규직=전부, 일용직=고용보험만, 프리랜서=없음). 직원명부의
4대보험 표기도 같은 jobType에서 유래하므로 두 화면이 어긋나면 급여대장에 경고 배너로 노출된다.

## 6. 검증

- `lib/payroll/simplified-tax-table.test.ts`: 별표2 공표 세액 1:1 대조, 부양가족 증가 시 감소,
  80% 비율, 자녀공제 차감, 범위 밖 처리.
- `lib/payroll-workspace/consistency.test.ts`: 고용형태별 정합/부정합 케이스.
- `lib/first-run-sample/seed.test.ts`: 정규직 소득세 조회 전환 후 급여 공제/실지급 합계 갱신.
