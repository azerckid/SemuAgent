# VAT AI Tax Treatment Rule Matrix
> Created: 2026-07-11
> Last Updated: 2026-07-11
> Backlog: JC-035 · VAI-2
> Rule Snapshot: `vat-kr-2026.07-v1`

## 0. Decision

이 문서는 SemuAgent가 부가세 거래를 **자동 확정하는 규칙**이 아니라, 사용자가 홈택스
자동채움 내용을 확인하고 수정할 항목을 찾기 위한 **판단 보조 규칙**을 고정한다.

화면의 중심 질문은 다음과 같다.

```text
홈택스 자동채움 예상과 SemuAgent의 확정 자료를 비교했을 때 무엇을 확인하거나 수정해야 하는가?
```

SemuAgent가 홈택스 신고 자료를 실제로 가져오지 않은 경우 `홈택스 현재값`이라고 표시하지
않는다. 이때는 `홈택스 자동채움 예상`과 `홈택스에서 확인할 항목`으로 표시한다. 향후 공식
홈택스 조회자료를 사용자가 업로드한 경우에만 실제 값 비교 상태를 별도 계약한다.

암호화 전자신고 파일은 범위 밖이다. 공식 엑셀·CSV 등 **양식 업로드**가 확인된 부속명세는
JC-030에서 별도로 다루며, JC-035는 홈택스 신고 화면에서 확인·수정할 세무 판단을 준비한다.

법령 개정 여부는 VAI-3 구현 시작 시 다시 확인한다. 새 규칙 버전은 기존 사용자 확정값을
몰래 바꾸지 않고 `재검토 필요`만 표시한다.

## 2. Required Facts

판단 전에 아래 사실을 같은 tenant·사업장·기간 범위에서 확인한다.

| 사실 | 필수 여부 | 부족할 때 |
|:---|:---:|:---|
| 거래 방향(`sale`/`purchase`) | 필수 | `needs_review` |
| 정확한 공급가액·세액·합계액과 산술 일치 | 필수 | VAT fact 보완 |
| 증빙 유형과 원천 행 | 필수 | 증빙 확인 |
| 거래처·품목·사용 목적 | 조건부 필수 | 내용 보완 또는 AI 해석 |
| 사업 관련성 | 매입 필수 | `needs_review` |
| 과세·면세 사업의 실지귀속 | 공통매입 필수 | 안분 검토 |
| 영세율·면세 법정 요건과 증빙 | 해당 거래 필수 | 확정 차단 |
| 같은 사업장의 과거 사용자 확정 | 선택 | 패턴 근거 없이 다음 단계 |

금액을 `합계액 / 11`로 추정하거나 거래처 업종만으로 영세율·면세·불공제를 확정하지 않는다.

## 3. Purchase Rule Matrix

| ID | 조건 | 추천 결과 | 홈택스 확인·수정 항목 | 필요한 근거·증빙 | 자동 확정 |
|:---|:---|:---|:---|:---|:---:|
| P-01 | 정확한 매입 VAT fact, 적격 증빙, 사업 관련성이 확인되고 불공제 규칙에 해당하지 않음 | `likely_deductible` | 매입세액 공제 상태 확인 | 세금계산서·카드전표·현금영수증 등과 업무 목적 | 금지 |
| P-02 | 개인 사용·가사 지출·사업 무관 사실이 명확함 | `likely_non_deductible` | 공제 상태라면 불공제로 변경 검토 | 사용 목적·소명·제외 근거 | 금지 |
| P-03 | 기업업무추진비·접대성 지출 가능성이 높음 | `likely_non_deductible` | 불공제 반영 여부 확인 | 거래처·참석자·업무 목적 | 금지 |
| P-04 | 비영업용 소형승용차의 구입·임차·유지 관련 가능성이 높음 | `likely_non_deductible` | 불공제 반영 여부 확인 | 차량 종류·용도·운행 사실 | 금지 |
| P-05 | 면세사업에 실지귀속되는 매입 | `likely_non_deductible` | 면세사업 관련 매입세액 불공제 확인 | 면세 매출과의 직접 귀속 근거 | 금지 |
| P-06 | 과세·면세 사업에 공통 사용되고 실지귀속을 구분할 수 없음 | `proration_required` | 공통매입세액 안분 항목 확인 | 귀속 기준·과세/면세 공급가액·안분율 근거 | 금지 |
| P-07 | 적격 증빙, 사업 관련성, 공급자 과세 여부 또는 금액 사실이 부족함 | `needs_review` | 공제·불공제 상태를 확정하지 않음 | 누락 사실 보완 | 해당 없음 |
| P-08 | 홈택스/국세청 사전 분류와 사용자 확정 이력이 다를 가능성이 있음 | 위 조건에 따른 추천 | `매입세액 공제 확인/변경` 대상 표시 | 현재 거래 사실과 과거 확정 근거 | 금지 |

P-01은 `공제 확정`이 아니라 `공제 가능성 높음`이다. 국세청 사전 분류도 최종 판단이
아니므로 사용자가 공제·불공제를 확인하거나 변경할 수 있어야 한다.

## 4. Sales Rule Matrix

| ID | 조건 | 추천 결과 | 홈택스 확인·수정 항목 | 필요한 근거·증빙 | 자동 확정 |
|:---|:---|:---|:---|:---|:---:|
| S-01 | 국내 과세 재화·용역 공급, 정확한 매출 VAT fact, 영세율·면세 신호 없음 | `likely_taxable` | 과세 매출 금액·세액 확인 | 세금계산서·카드·현금영수증·계약 | 금지 |
| S-02 | 수출 등 영세율 가능 거래이며 법정 요건과 증빙을 확인할 수 있음 | `likely_zero_rated` | 영세율 매출과 첨부서류 확인 | 수출신고·외화입금·내국신용장·구매확인서 등 해당 증빙 | 금지 |
| S-03 | 법정 면세 재화·용역 가능성이 있고 실질 거래 정보가 있음 | `likely_exempt` | 면세 매출 구분과 수입금액 확인 | 품목·용역 내용·인허가·면세 요건 | 금지 |
| S-04 | 계좌이체·핀테크·현금 매출 등 홈택스 자동채움 누락 가능성이 있음 | `needs_review` | 기타 매출 누락 여부와 금액 추가·수정 확인 | 통장·정산내역·현금매출 근거 | 해당 없음 |
| S-05 | 영세율·면세 명칭만 있고 필수 사실 또는 증빙이 부족함 | `needs_review` | 과세유형을 확정하지 않음 | 부족한 사실·증빙 목록 | 해당 없음 |
| S-06 | 비과세/신고대상 제외 가능성이 있으나 공식 규칙으로 단정할 수 없음 | `needs_review` | 전문가 확인 대상으로 표시 | 거래 구조·계약·법적 근거 | 해당 없음 |

영세율·면세는 `가능성`만으로 사용자 확정 또는 downstream gate를 통과할 수 없다. 필수
증빙이 `present`이고 사용자가 명시적으로 확인한 경우에만 최종 과세유형에 반영한다.

## 5. Hometax Action Contract

| 코드 | 화면 라벨 | 의미 |
|:---|:---|:---|
| `expected_no_change` | 그대로 확인 | SemuAgent 판단과 홈택스 자동채움 예상이 일치할 가능성이 높음 |
| `review_deduction` | 공제·불공제 확인 | 홈택스의 매입세액 공제 상태를 확인하거나 변경해야 함 |
| `review_sales_tax_type` | 과세유형 확인 | 과세·영세율·면세 구분을 확인하거나 변경해야 함 |
| `add_or_correct_amount` | 금액 추가·수정 확인 | 자동채움 누락·중복·취소·현금/계좌 매출 등을 확인해야 함 |
| `review_proration` | 안분 확인 | 공통매입세액 안분 근거와 비율을 확인해야 함 |
| `compare_in_hometax` | 홈택스에서 비교 | 실제 홈택스 값이 없으므로 신고 화면과 직접 대조해야 함 |

`expected_no_change`도 신고 확정이나 자동 제출을 뜻하지 않는다. 실제 홈택스 자료를 가져오지
않은 동안 모든 행의 비교 기준은 `expected_prefill`이며, 화면에 실제 차액처럼 표시하지 않는다.

## 6. Evidence Codes

| 코드 | 의미 | 대표 적용 |
|:---|:---|:---|
| `exact_vat_fact` | 공급가액·세액·합계액·과세유형 산술 일치 | 모든 매출·매입 |
| `qualified_purchase_evidence` | 적격 매입 증빙 | P-01~P-07 |
| `business_purpose` | 사업 관련 사용 목적 | P-01~P-04 |
| `vehicle_business_use` | 차량 종류·영업용 여부·업무 사용 | P-04 |
| `taxable_exempt_attribution` | 과세·면세 사업 실지귀속 | P-05~P-06 |
| `proration_basis` | 안분 기준과 산식 | P-06 |
| `export_or_zero_rate_documents` | 영세율 법정 증빙 | S-02 |
| `exemption_qualification` | 면세 품목·용역·인허가·실질 요건 | S-03 |
| `unreported_sales_basis` | 계좌·핀테크·현금 등 누락 매출 근거 | S-04 |

증빙 상태는 `present`, `missing`, `needs_review`만 사용한다. `missing`인 필수 증빙은
영세율·면세 확정과 VAT gate 해제를 차단한다.
