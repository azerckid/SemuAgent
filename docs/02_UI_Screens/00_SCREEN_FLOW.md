# JARYO Company Screen Flow
> Created: 2026-07-01 19:40
> Last Updated: 2026-07-02

## 1. 범위

이 문서는 v1 화면의 사용자 흐름을 정의한다. 현재 정의된 화면:
1. **회사 홈(Company Home)** — 승인 완료(2026-07-01)
2. **자료수집(Source Collection)** — 승인 완료(2026-07-01)
3. **기장검토(Bookkeeping Review)** — 승인 완료(2026-07-01)
4. **부가세(VAT)** — 승인 완료(2026-07-01)
5. **급여(Payroll)** — 승인 완료(2026-07-01)
6. **신고지원(Filing Support)** — 승인 완료(2026-07-01)

6개 워크스페이스 전체 프리뷰 승인 완료(6/6 UI 확정, 2026-07-01).

외부 연동(고객 업로드 포털 `/upload/[token]`, 외부 세무사 검토 흐름)은 v1 범위에서 제외한다.
회사 내부 운영 화면만 다룬다.

## 2. 전체 여정 (Company Home 중심)

```
로그인
  └─▶ 회사 홈 (첫 진입 = 대시보드, 마케팅 페이지 아님)
        ├─ 현재 회계기간 상태 확인
        ├─ "다음 할 일"(신고 전 blockers)에서 액션 선택
        │     ├─▶ 기장검토 (미분류 거래)      [구현 완료]
        │     ├─▶ 자료수집 (미수집 자료)        [구현 완료]
        │     └─▶ 부가세 (공제 검토)            [다음 단계에서 구현]
        ├─ 준비 현황 카드 클릭 → 해당 워크스페이스로 이동
        └─ 최근 제출·영수증 → 전체 보기
```

## 3. 진입 · 전환 · 이탈

| 구분 | 내용 |
|:---|:---|
| 진입점 | 로그인 성공 직후 회사 홈으로 리다이렉트. 사이드바 "회사 홈" 클릭. |
| 핵심 CTA | ① 다음 할 일의 "…열기" 버튼 → 해당 워크스페이스. ② 준비 현황 카드 클릭 → 워크스페이스. ③ 상단 기간 선택 → 다른 회계기간으로 컨텍스트 전환. |
| 정상 이탈 | 워크스페이스로 이동하여 실제 작업 수행. |
| 이상 이탈 | 데이터 로드 실패 시 오류 상태 → "다시 시도". 자료 0건이면 빈 상태 → "첫 자료 업로드". |

## 4. 화면별 데이터 입출력 (회사 홈)

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 회계기간 상태 | 선택된 tenant·기간 | 진행률, 마감일, D-day | 없음(읽기) |
| 다음 할 일 | 미분류 거래 수, 미수집 자료, 공제 검토 수 | 액션 목록 + CTA | 없음(읽기, 클릭 시 라우팅) |
| 준비 현황 카드 | 워크스페이스별 집계 | 건수·상태칩 | 없음(읽기) |
| 최근 제출·영수증 | 제출/영수증 레코드 | 표(구분·항목·기간·상태·일시) | 없음(읽기) |
| 기간 선택 | 사용자 선택 | 활성 기간 | 세션/URL 컨텍스트 갱신 |

회사 홈은 **읽기 전용 대시보드**이다. 데이터 변경(mutation)은 각 워크스페이스에서 수행한다.

## 4b. 자료수집 화면 흐름 및 데이터

진입: 회사 홈 사이드바 "자료수집" 또는 "다음 할 일"의 "자료수집 열기" → 자료수집 화면.
복귀: 사이드바 "회사 홈" → 대시보드.

```
회사 홈 ──"자료수집 열기" / 사이드바──▶ 자료수집
   ▲                                      │
   └──────────── 사이드바 "회사 홈" ───────┘
```

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 수집 완결성 | tenant·기간, 자료 집계 | 진행률, 미수집 건수 | 없음(읽기) |
| 파일 업로드 | 사용자 파일(XLSX/CSV/PDF/이미지/ZIP) | 드롭존, 지원 형식 안내 | 업로드 → 파싱/정규화 큐 등록 |
| 자료유형 정규화 | 파싱 결과 | 유형별 건수·상태칩 | 정규화 확정/재분류 |
| 수집(가져오기) 상태 | 업로드 레코드 | 파일·유형·진행·상태·일시 | 재시도, 상세 보기 |
| 미수집·확인 필요 | 필수 자료 대비 부족분 | 항목 목록 + CTA | 재업로드/정규화 확인으로 라우팅 |

자료수집은 **업로드·정규화 mutation이 발생**하는 화면이다(회사 홈과 달리 읽기 전용 아님).
외부 고객 업로드 포털은 제외하고, 회사 내부 직접 업로드만 다룬다.

## 4c. 기장검토 화면 흐름 및 데이터

진입: 회사 홈 "다음 할 일"의 "기장검토 열기"(미검토 거래 분류) 또는 사이드바 "기장검토" → 기장검토 화면.
복귀: 사이드바 "회사 홈" 등으로 이동.

```
회사 홈 ──"기장검토 열기" / 사이드바──▶ 기장검토
   │                                        │ 거래 승인/수정
   │                                        ▼
   └──────────── 사이드바 ────────── 확정 전표(확정 탭)
```

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 분류 현황 | tenant·기간, 거래 집계 | 확정/대기 건수, 진행률 | 없음(읽기) |
| 분류 큐 | 정규화된 거래, AI 추천 계정과목·신뢰도 | 거래일·내용·상대처·금액·추천계정·신뢰도 | 계정과목 확정/변경 |
| 선택 거래 상세 | 선택 거래 | 분개 미리보기(차/대변), 기간 귀속, 증빙, 부가세 공제 | 없음(읽기, 승인 전 미리보기) |
| 승인 | 사용자 승인 액션 | 상태칩(검토 대기→확정) | 전표 확정(mutation), 확정 탭으로 이동 |
| 일괄 처리 | 다중 선택 | 선택 건수 | 일괄 승인/계정 변경(mutation) |

기장검토는 **분류 확정·승인 mutation이 발생**한다. AI 추천은 초안이며 확정 책임은 사용자에게 있다.
신뢰도 낮은 항목(예: 접대비/복리후생비 경계)은 "계정 지정"으로 강제 확인시킨다.

## 4d. 부가세 화면 흐름 및 데이터

진입: 회사 홈 "다음 할 일"의 "부가세 열기"(공제 검토) 또는 사이드바 "부가세" → 부가세 화면.
선행: 기장검토에서 거래가 확정되어야 매출·매입 집계가 정확해진다.

```
기장검토(확정 전표) ──집계──▶ 부가세
회사 홈 ──"부가세 열기" / 사이드바──▶ 부가세 ──검토 3건 완료──▶ 신고 패키지 생성
```

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 세액 요약 | 확정 전표 집계 | 매출세액 − 매입세액 = 납부(예정)세액, 마감 D-day | 없음(읽기) |
| 매출 구분 | 과세/영세율/면세 매출 | 그룹별 공급가액·매출세액 | 없음(읽기) |
| 매입세액 공제 검토 | 매입 거래, 불공제/안분 후보 | 공급가액·세액·공제 판정·사유 | 공제/불공제 확정, 공통매입 안분(mutation) |
| 부속 명세 | 합계표·수취명세서 등 | 서식별 준비 상태 | 없음(읽기, 패키지에 포함) |
| 신고 패키지 | 확정된 세액·명세 | 패키지 미리보기(PDF), 홈택스 입력 가이드 | 패키지 생성(mutation) |

부가세는 **공제 판정·안분·패키지 생성 mutation이 발생**한다. 불공제 후보 검토가 끝나야 최종 패키지를 생성할 수 있다.
**자동 홈택스 제출은 범위 밖**(신고 패키지 + 입력 가이드까지). 외부 세무사 검토 흐름은 v1 제외.

## 4e. 급여 화면 흐름 및 데이터

진입: 사이드바 "급여" → 급여 화면. 월별 급여 기간 선택.
연계: 원천징수 지급명세서 등은 **신고지원** 화면으로 전달된다.

```
급여 입력/자료 ──계산──▶ 급여대장 ──확인 필요 처리──▶ 급여 마감·확정
                                                        │ 지급명세서
                                                        ▼
                                                     신고지원
```

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 급여 요약 | 직원별 급여 입력·계산 | 지급총액·공제총액·실지급액·마감상태 | 없음(읽기) |
| 확인 필요 알림 | 오류/누락 직원(취득일·계좌 등) | 사유 + CTA | 해당 직원 편집으로 라우팅 |
| 급여대장 | 직원별 기본급·수당·공제 | 지급/원천세/4대보험/공제계/실지급 표 | 엑셀 내보내기, 개별 편집(mutation) |
| 공제 상세 | 원천세·4대보험 요율 | 항목별 공제 집계 | 없음(읽기) |
| 명세서·마감 | 확정된 급여 | 급여명세서·지급명세서 미리보기, 마감 버튼 | 명세서 생성, 급여 마감·확정(mutation) |

급여는 **개별 편집·명세서 생성·마감 확정 mutation이 발생**한다. 확인 필요(오류/누락) 직원이 있으면 마감 전 처리해야 한다.
개인정보(급여·주민정보)를 다루므로 접근 권한과 마스킹은 구현 단계에서 확정한다. 외부 세무사 흐름은 v1 제외.

## 4f. 신고지원 화면 흐름 및 데이터

진입: 사이드바 "신고지원" 또는 각 신고 항목의 "…열기". 부가세·급여 화면에서 생성된 자료가 신고 항목으로 모인다.

```
부가세 패키지 ─┐
급여 지급명세서 ─┼─▶ 신고지원(항목·패키지) ─입력 가이드─▶ (회사가 홈택스 직접 제출) ─▶ 접수증 보관 ─▶ 사후 체크리스트
4대보험 자료 ──┘
```

책임 경계: **자동 제출·자동 납부 없음.** 첨부 패키지 생성 + 홈택스 단계별 입력 가이드 + 접수증 보관 + 사후 체크리스트까지.

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 신고 항목·패키지 | 부가세·급여·4대보험 산출물 | 항목별 상태(준비됨/대기/확인 필요)·패키지 | 패키지 생성(mutation), 선행 화면으로 라우팅 |
| 홈택스 입력 가이드 | 확정된 신고 값 | 단계별 입력 값(인원·지급액·세액) | 없음(읽기, 값 복사) |
| 제출 접수증 보관 | 사용자 업로드 접수증 | 접수증 목록·보관 상태 | 접수증 업로드·보관(mutation) |
| 사후 체크리스트 | 납부·보관 진행 | 체크 항목 완료 상태 | 체크 상태 갱신(mutation) |

신고지원은 **패키지 생성·접수증 보관·체크리스트 갱신 mutation이 발생**한다. 실제 제출/납부는 회사가 홈택스에서 직접 수행한다.

## 5. 상태 커버리지

여섯 화면 공통 상태 규약. 각 화면 Preview 하단 "화면 상태 예시"에 로딩/빈/오류를 명시한다.

| 상태 | 회사 홈 | 자료수집 | 기장검토 | 부가세 | 급여 | 신고지원 |
|:---|:---|:---|:---|:---|:---|:---|
| Default | 대시보드 집계 | 수집 상태 | 분류 큐 | 세액·공제 검토 | 급여대장 | 신고 항목·접수증 |
| Loading | 카드·표 스켈레톤 | 동일 | 동일 | 동일 | 동일 | 동일 |
| Empty | "첫 자료 업로드" | "첫 자료 업로드" | "확정 전표 보기" | "기장검토 먼저 확정" | "급여 자료 불러오기" | "부가세·급여 먼저 확정" |
| Error | "현황을 불러오지 못했습니다" | "파일을 처리하지 못했습니다" | "분류 큐를 불러오지 못했습니다" | "세액 집계를 불러오지 못했습니다" | "급여 계산을 불러오지 못했습니다" | "신고 항목을 불러오지 못했습니다" |
| 권한 없음 | tenant 미소속/미인증 시 접근 차단 (구현 단계에서 확정) | 동일 | 동일 | 동일 | 동일(개인정보 마스킹) | 동일 |
