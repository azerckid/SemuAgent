## 1. 범위

이 문서는 v1 화면의 사용자 흐름을 정의한다. 현재 정의된 화면:
1. **회사 홈(Company Home)** — 승인 완료(2026-07-01)
2. **자료수집(Source Collection)** — 승인 완료(2026-07-01)
3. **기장검토(Bookkeeping Review / 자료대조원장)** — 승인 완료(2026-07-01), Path 1 데이터 확정 관문 위치로 정리(2026-07-08)
4. **급여·지급(Payroll & Payments)** — 기존 급여 화면과 원천세·지급명세서·지방소득세·직원 명부를 묶는 월간 업무 그룹
5. **부가세(VAT)** — 승인 완료(2026-07-01), 분기·반기 업무 그룹
6. **원천세(Withholding Tax)** — 기존 신고지원의 원천세 준비값을 급여·지급 하위로 재배치
7. **직원 명부(Employee Directory)** — 구현 완료(2026-07-02, 급여·지급 기준정보·JC-015)
8. **리마인드(Internal Reminder)** — 구현 완료(2026-07-02, 운영 보조 화면·JC-016)
9. **연간신고(Annual Filing)** — 기존 신고 준비 허브를 사업자 유형별 연간 세목 작업공간으로 재구성
10. **지급명세서·연말정산(Payment Statement · Year-end)** — 구현 완료(2026-07-05, 급여·지급 하위)
11. **지방소득세(Local Income Tax)** — 구현 완료(2026-07-05, 원천세 특별징수분 한정, 급여·지급 하위)
12. **사업장현황신고(Business Status Report)** — 구현 완료(2026-07-05, 면세 개인사업자에게만 연간신고 하위 노출)

2026-07-11 IA 결정에 따라 상위 메뉴 `신고지원`과 `신고 준비`는 제거한다. 사용자가 실제로 기억하는 업무 주기에 맞춰 `급여·지급`(월), `부가세`(분기·반기), `연간신고`(연)로 재배치한다. 직원 명부는 급여·지급 하위 기준정보이며, 신고 일정 요약은 별도 허브가 아니라 회사 홈의 `다가오는 신고` 스트립으로 제공한다. 상세 계약은 [Cadence Navigation Prototype Review](./13_CADENCE_NAVIGATION_PROTOTYPE_REVIEW.md)를 따른다.

외부 연동(고객 업로드 포털 `/upload/[token]`, 외부 세무사 검토 흐름)은 v1 범위에서 제외한다.
회사 내부 운영 화면만 다룬다.


| 준비 현황 카드 | 워크스페이스별 집계 | 건수·상태칩 | 없음(읽기) |

## 4c. 기장검토 화면 흐름 및 데이터

진입: 회사 홈 "다음 할 일"의 "자료대조원장 열기"(미검토 거래 분류) 또는 사이드바 "기장검토 > 자료대조원장" → `/dashboard/bookkeeping/reconciliation-ledger`. 사이드바 "기장검토" → `/dashboard/bookkeeping` 분류 큐.
복귀: 사이드바 "회사 홈" 등으로 이동.

위치 결정(2026-07-08): 자료를 모두 올린 뒤 세목별 양식 생성 전에 확인하는 첫 관문은 신고 준비 하위가 아니라 **기장검토 하위의 "자료대조원장"** 으로 둔다. 신고 준비는 이 관문을 통과한 확정 데이터를 세목별 양식·파일 준비 상태로 보여주는 마무리 허브다.

```
회사 홈 ──"자료대조원장 열기"──▶ 기장검토 > 자료대조원장 (`/dashboard/bookkeeping/reconciliation-ledger`)
   │                                        │ 거래 승인/수정
   │                                        ▼
   └──────────── 사이드바 ────────── 확정 전표(확정 탭)
```

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 기장검토 분류 큐 | 정규화된 거래, AI 추천 계정과목·신뢰도 | 거래일·내용·상대처·금액·추천계정·신뢰도 | 계정과목 확정/변경 |
| 자료대조원장 요약 | 통장·카드·세금계산서·현금영수증·수기 자료 | 원장 준비율, 확정/대기/증빙·소명 필요/제외 검토 건수 | 없음(읽기) |
| 자료대조 원장 표 | 출처별 거래 + 연결 증빙 | 출처·거래처·적요·공급가액·세액·연결 증빙·계정·거래처·상태 | 증빙 연결/계정 지정/확정/제외 |
| 선택 거래 상세 | 선택 거래 | 분개 미리보기, 기간 귀속, 증빙, 부가세 공제, 제외 사유 | 승인 전 미리보기 |
| 세목별 생성 가능 여부 | 확정 거래원장 + tax-specific blockers | 부가세·사업장현황신고·지방소득세 등 Path 1 파일 생성 가능 여부 | 없음(읽기) |

기장검토는 **분류 확정·승인 mutation이 발생**한다. AI 추천은 초안이며 확정 책임은 사용자에게 있다.
신뢰도 낮은 항목(예: 접대비/복리후생비 경계)은 "계정 지정"으로 강제 확인시킨다. 자료대조원장은 별도 세목 화면이 아니라 기장검토 하위의 Path 1 관문이다. 기존 기장검토 분류 큐와 달리 통장·카드·세금계산서·현금영수증을 같은 원장 표에서 대조하고, 홈택스 업로드용 양식·파일 생성 전에 증빙 연결·계정항목·거래처·업무무관 제외 여부를 확정한다. 단순히 후보 개수를 보여주는 것은 완료 상태가 아니며, AI가 찾은 실제 증빙 행을 보여주거나 사용자가 직접 증빙을 찾게 해서 연결·해제·계정 지정·소명·제외를 끝낼 수 있어야 한다. 기간 단위는 월/분기/반기/연/사용자 지정으로 전환 가능해야 하며, 기본값은 진입한 신고 맥락(예: 원천세 월, 부가세 1기, 간이지급 반기, 사업장현황신고 연)에 맞춘다. "증빙없음"은 완료 상태가 아니라 내부 원인/문제이므로 화면에는 증빙 필요·소명 필요·소명 완료·증빙 예외·제외됨처럼 사용자가 해야 할 행동과 해결 상태로 표시한다. 또한 전월/최근 확정 패턴을 학습해 계정·증빙·제외 추천의 근거로 보여줄 수 있지만, 사용자의 확인 없이 자동 확정하지 않는다. 기본 사용 흐름은 다음 할 일 큐에서 세목 파일 생성을 막는 항목부터 처리하고, 원장 행의 증빙 상태·계정항목·한 줄 결론 셀에서 계정·증빙·소명·제외 추천을 빠르게 판단한 뒤, 필요한 경우 모달을 열어 안전한 반복 패턴만 일괄 수락하는 방식이다. 부족한 원천 자료는 자료수집으로 되돌아가는 링크를 제공하고, 최근 적용은 얕은 undo로 되돌릴 수 있어야 한다. 현재 앱 화면은 1차 slice이며, 통장-세금계산서 매칭, 카드-증빙 연결, 소명/사용내역 메모, AI 계정 추천 신뢰도 표시, 사적 사용 의심, 제외 사유 선택, 인라인 계정 수정이 완성되어야 자료대조원장 역할을 충족한다. 입출금 대조 추천, 사용내역 소명 모달, 제외 사유 taxonomy, Path 1 blocker 계약은 [Reconciliation Ledger Phase 2 Pre-Code Brief](../03_Technical_Specs/41_RECONCILIATION_LEDGER_V2_PRE_CODE_BRIEF.md)를 따른다.

## 4d. 부가세 화면 흐름 및 데이터

진입: 회사 홈 "다음 할 일"의 "부가세 열기"(공제 검토) 또는 사이드바 "부가세" → 부가세 화면.
선행: 기장검토에서 거래가 확정되어야 매출·매입 집계가 정확해진다.

```
기장검토(확정 전표) ──집계──▶ 부가세
회사 홈 ──"부가세 열기" / 사이드바──▶ 부가세 ──AI 판단·사용자 확정──▶ 확정 원장 다시 계산 ──fingerprint 확인──▶ 신고 준비 검토 자료
```

JC-035의 부가세 판단 흐름은 다음 순서다.

```text
exact VAT fact
  -> 공식 규칙
  -> 같은 사업장 이전 확정 패턴
  -> 애매한 행만 단일 AI
  -> 고위험·모델 불일치만 multi-provider 검토
  -> 영세율·면세 필수 증빙 사용자 확인 완료
  -> 사용자 확정 또는 전문가 확인
```

AI는 공제/불공제/안분과 과세/영세율/면세의 **가능성·근거·필요 증빙**을 표시한다.
각 행에는 `홈택스 자동채움 예상` 기준으로 그대로 확인할지, 공제·불공제/과세유형/금액/안분을
확인하거나 수정할지를 함께 표시한다. 실제 홈택스 조회자료를 가져오지 않은 상태에서는
`홈택스 현재값`이나 실제 차액이라고 단정하지 않는다.
영세율·면세는 필요한 증빙이 없으면 확정할 수 없다. 사용자는 판단 행의 필수 증빙을
실제로 준비했을 때만 `확인 완료`로 기록하며, 확인 취소 시 해당 gate가 다시 잠긴다.
확인 기록은 거래·증빙 코드·확인자·확인시각 범위로 감사한다. AI timeout·quota·provider 오류가
발생해도 화면은 멈추지 않고 해당 행만 `수동 확인 필요`로 전환한다. 사용자의 명시적
확정 전에는 VAT fact·공제 decision·세액·package gate를 변경하지 않는다.
사용자는 각 행에서 `적용`·`다르게`·`보류`·`전문가 확인`을 선택하고, 방금 저장한
가장 최근 작업은 toast의 `되돌리기`로 한 번 복원할 수 있다. 되돌리기는 일회용 토큰과
현재 canonical 상태를 서버에서 다시 확인하며, 이후 값이 바뀌었으면 실패 안전하게 거부한다.

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 세액 요약 | 확정 전표 집계 | 매출세액 − 매입세액 = 납부(예정)세액, 마감 D-day | 없음(읽기) |
| 매출 구분 | 과세/영세율/면세 매출 | 그룹별 공급가액·매출세액 | 없음(읽기) |
| 확인 필요 거래 | VAT 판단 행 + 매입 공제 검토 | 예외 거래 4열 작업대, 근거·증빙·홈택스 상세 펼치기 | 세무판단 적용/다르게/보류/전문가 확인 + 공제/불공제/안분 mutation |
| 확정 원장 재계산 | 같은 tenant/사업장/반기의 최신 확정 VAT fact + 연결된 공제 검토 | stale/verified/blocked provenance 상태 | 모든 다른 gate가 준비된 stale 상태에서만 사용자가 명시적으로 재계산(mutation) |
| compact 신고 준비 | canonical gate·차단 이유 | 한 줄 상태 + 차단 이유 펼치기; 대형 package/부속명세 카드 없음 | 신고 준비 완료 처리(mutation) |

부가세는 **공제 판정·안분·사용자 세무판단 확정·확정 원장 재계산 mutation이 발생**한다. 같은 classification 행의 세무판단과 공제 검토는 한 작업대에 합치며, 사용자 확정 완료 또는 확정 fact·공식 규칙·필수 증빙이 모두 일치한 정상 행은 기본 목록에서 제외한다. 자료수집·자료대조·공제 검토가 끝나고 deterministic fingerprint가 현재 입력과 일치해야 신고 준비를 마감할 수 있다. 재계산 가능한 경우에만 `확정 원장 다시 계산`을 노출하며 자동으로 실행하지 않는다. 부가세 전체 신고용 공식 비암호화 업로드 파일은 JC-030 Stage A 외부 확인 전에는 제공하지 않는다.
**자동 홈택스 제출은 범위 밖**(신고 준비값 확인까지). 외부 세무사 검토 흐름은 v1 제외.

## 4e. 급여·지급 화면 흐름 및 데이터

진입: 사이드바 "급여·지급" → 월별 급여 작업공간. 직원 명부·원천세·지급명세서·연말정산·지방소득세는 같은 그룹의 하위 화면으로 이동한다.

```
직원 명부 + 급여 입력/자료 + 4대보험 고지내역 ──계산/매칭──▶ 급여대장 ──마감·확정──▶ 원천세·지방소득세·지급명세서
```

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 급여 요약 | 직원별 급여 입력·계산, 고지액 매칭 | 지급총액·공제총액·실지급액·마감상태 | 없음(읽기) |
| 확인 필요 알림 | 오류/누락 직원(취득일·보험료 고지 매칭 등) | 사유 + CTA | 해당 직원 편집으로 라우팅 |
| 급여대장 | 직원별 기본급·수당·공제 | 지급/원천세/4대보험/공제계/실지급 표 | 엑셀 내보내기, 개별 편집(mutation) |
| 공제 상세 | 원천세·4대보험 요율, 건강보험 EDI/사회보험 고지내역 | 항목별 공제 집계, 고지액 반영 여부 | 고지내역 업로드/수동 입력·매칭(mutation) |
| 명세서·마감 | 확정된 급여 | 급여명세서·지급명세서 미리보기, 마감 버튼 | 명세서 생성, 급여 마감·확정(mutation) |

급여는 **개별 편집·명세서 생성·마감 확정 mutation이 발생**한다. 확인 필요(오류/누락) 직원이 있으면 마감 전 처리해야 한다.
건강보험 EDI/사회보험 고지내역은 업로드 또는 수동 입력으로 반영하며, 자동 로그인·공동인증서 저장·자동 제출은 v1 제외다.
개인정보(급여·주민정보)를 다루므로 접근 권한과 마스킹은 구현 단계에서 확정한다. 외부 세무사 흐름은 v1 제외.

## 4f. 원천세 화면 흐름 및 데이터 (기존 신고지원 재배치)

진입: 사이드바 `급여·지급 > 원천세`. 확정 급여에서 원천징수 인원·총지급액·소득세·지방소득세를 읽고, 홈택스 신고 경로·기본정보·A01 행/칸 위치와 값을 한 표로 정리한다.

```text
급여 마감·확정 ──▶ 원천세 준비값 ──사용자 확인──▶ 사용자가 홈택스에 직접 입력·제출
```

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 원천세 홈택스 입력표 | 확정 급여·지급내역 | 신고 경로, 신고구분, 귀속연월, 지급연월, 사업자, A01 ④⑤⑥, blocker | 없음(읽기) |
| 위택스 별도 값 | 확정 지방소득세 | 특별징수분 합계, 홈택스 합산 금지 안내 | 없음(읽기) |
| 제출 접수증 | 사용자 업로드 접수증 | 접수증 목록·보관 상태 | 업로드·보관(mutation) |

기존 신고지원의 부가세·4대보험 패키지와 혼합 체크리스트는 제거한다. 부가세 내용은 부가세 화면으로, 연간 세목은 연간신고로 이동한다. 상위 `신고지원` 메뉴는 더 이상 사용하지 않는다.

## 4h. First-run Sample Data 흐름 및 데이터

목적: 신규 사용자가 가입 직후 빈 화면 대신 승인 Preview와 유사한 샘플 업무 상태를 보고 메뉴 구조를 이해한다.

```
온보딩 완료 ──sample seed──▶ 회사 홈/각 워크스페이스(sample active)
       │                              │
       │                              └─▶ "샘플 데이터 삭제하고 실제 사용 시작"
       │                                             │
       └────────────── seed 실패 시 재시도 안내 ◀─────┘
```

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 샘플 생성 | 신규 tenant·사업장 | 승인 Preview와 유사한 채워진 상태 | sample dataset + domain row seed |
| 샘플 표시 | active sample dataset | 전역 banner, 샘플 badge, 삭제 CTA | 없음(읽기) |
| 샘플 삭제 | 사용자 확인 | 삭제 중/완료/실패 상태 | registry 기반 샘플 row 삭제 |
| 샘플 실패 | seed/delete 오류 | 재시도 CTA | 오류 사유 저장 |

샘플 데이터는 **학습용**이며 실제 신고 자료가 아니다. 삭제는 registry가 추적하는 샘플 행만 대상으로 하며, 실제 업로드·급여·신고 데이터는 삭제하지 않는다.

## 4g. 연간신고 화면 흐름 및 데이터 (기존 JC-029 재구성)

진입: 사이드바 `연간신고`. 사업자 유형에 따라 법인은 **법인세**, 일반 개인은 **종합소득세**, 면세 개인은 **종합소득세 + 사업장현황신고**만 표시한다. 기존 신고 준비 허브처럼 월·분기 세목까지 한 화면에 나열하지 않는다.

연계: 확정 거래원장과 연간 급여·지급 자료를 읽어 연간 세목의 준비 상태와 blocker를 보여준다. Path 1a/1b 경계는 유지한다.

- **Path 1a (양식 있음):** 홈택스가 직접 수용하는 공식 비암호화 양식 파일 생성 + 업로드 안내 (JC-030·JC-013)
- **Path 1b (양식 없음):** 홈택스 메뉴 경로와 신고서의 행·칸 위치에 확정값을 연결한 직접입력 화면 제공 — 사용자가 해당 위치에 직접 입력 (자동입력·자동제출·클릭별 화면 캡처 안내는 제외)
- **Path 2 (후순위):** 사무소 handoff ZIP (JC-034, 구현 보류)
- **Path 3:** 인증·암호화 파일은 현재 제품 범위 밖

핵심 경계: 이 화면은 제출 버튼이 아니다. SemuAgent는 신고 데이터와 제출 보조 자료를 준비하고, 최종 신고·제출·납부는 사용자가 직접 수행한다. 자동제출은 JC-023 별도 결정 없이는 도입하지 않는다.

    자료수집 -> 기장검토 -> 확정 거래원장
                             └─ 연간신고
                                  ├─ 법인: 법인세
                                  ├─ 개인: 종합소득세
                                  └─ 면세 개인: 사업장현황신고
                                       -> Path 1a: 양식 있으면 파일 + 업로드 안내
                                       -> Path 1b: 양식 없으면 항목=값 직접입력 정리

구성:
- 상단 = 사업자 유형, 귀속연도, 연간신고 준비율과 blocker
- 본문 = 해당 사업자에게 필요한 연간 세목만 표시
- 하단 = Path 1a/1b 준비 상태와 직접 신고 책임 경계

| 사업자 유형 | 노출 세목 | 입력 | handoff |
|:---|:---|:---|:---|
| 법인 | 법인세 | 확정 원장·재무 집계 | Path 1 계약은 JC-026에서 확정 |
| 일반 개인 | 종합소득세 | 확정 원장·소득/경비 집계 | Path 1 계약은 JC-025에서 확정 |
| 면세 개인 | 종합소득세·사업장현황신고 | 면세 수입·경비·사업자 유형 | 사업장현황신고 Path 1b 화면 구현 예정 |

1차 구현 범위는 기존 트랙을 재프레임하고 메뉴/화면명을 바꾸는 것이다. 신규 산출 엔진, 신규 DB, 세액 계산, 자동제출은 이번 JC-029 범위 밖이다.

## 4i. 지급명세서·연말정산 준비 검토 화면 흐름 및 데이터 (JC-024)

진입: `급여·지급 > 지급명세서·연말정산` → 전용 검토 화면.
연계: 급여(JC-012)의 월별 원천·지급 데이터와 직원 명부(JC-015)를 반기/연 단위로 집계한다. 확인 필요(누락)는 급여·직원 명부로 라우팅한다.

핵심 경계: 이 화면은 **신고 준비 데이터**를 정리·검토하는 read-only 집계 화면이다. 정산액(결정세액·환급/추징) 계산·홈택스 제출은 범위 밖이다. **JC-030 전자신고 파일 생성 패널**(UI Preview)은 동일 화면에서 파일 후보·사전검증·다운로드 안내까지 제공하며, 홈택스 업로드·제출은 사용자가 직접 한다. 자동 제출·자격증명 저장은 하지 않는다.

구성(단일 스크롤·직원 중심):
- 상단 = 귀속연도/반기 선택, 대상 인원·확인 필요(누락)·데이터 준비 완료 수
- 확인 필요 = 급여 미확정·인적사항 누락 → 급여/직원 명부 라우팅
- 간이지급명세서(근로소득·반기) = 직원별 귀속기간·지급총액·원천징수세액·준비 상태
- **전자신고 파일 생성(JC-030)** = 데이터·공식 비암호화 양식 상태·4단계 흐름·사전검증·PII 1회 입력·홈택스 직접 업로드 안내
- 연말정산 준비·검토 = 직원별 재직·연간 지급합계·기납부 원천세·누락·검토 상태(정산액 계산은 범위 밖)
- 책임 경계

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 준비 요약 | tenant·기간, 급여/직원 집계 | 준비율·대상 인원·누락 수 | 없음(읽기) |
| 확인 필요 | 급여 미확정·인적사항 누락 | 사유 + CTA | 없음(읽기, 급여/직원 명부로 라우팅) |
| 간이지급명세서(근로·반기) | 월별 payroll 원천 집계 | 직원별 지급총액·원천징수세액·상태 | 없음(읽기 집계) |
| 연말정산 준비·검토 | 직원별 연간 지급·기납부 | 연간 합계·기납부 원천세·누락·검토상태 | 없음(읽기 집계) |

지급명세서·연말정산은 **read-only 집계 화면**이다(mutation 없음). v1은 간이지급명세서(근로소득)·연말정산 데이터·검토까지이며, 정산액 계산·전자신고 파일 생성·자동제출은 범위 밖이다.

## 4j. 지방소득세 준비 화면 흐름 및 데이터 (JC-027)

진입: `급여·지급 > 지방소득세` → 원천세 특별징수분 검토 화면.
연계: 급여(JC-012)에 이미 기록된 `payrollEmployeeLine.localIncomeTaxKrw`(원천세 특별징수분)를 원천세 신고 주기 기준으로 집계한다. 확인 필요(급여 미확정)는 급여로 라우팅한다.

핵심 경계: 이 화면은 **원천세 특별징수분 지방소득세만** 다룬다(종합소득세분·법인세분은 범위 밖, JC-025/026 이후). 실제 기록된 값을 집계할 뿐 **신규 세액 계산이 아니다**(근사 계산 아님). 신고지원(JC-013)이 표시하던 근사치(`splitWithholdingTax`)를 이 화면과 같은 실제값으로 교체해, 두 화면이 같은 기간에 다른 숫자를 보이지 않게 한다. 위택스 제출은 사용자가 직접 수행하며 자동 제출·자격증명 저장은 하지 않는다.

구성(단일 스크롤·직원 중심):
- 상단 = 귀속기간 선택(원천세 신고 주기 기준), 대상 인원·확인 필요·지방소득세 합계
- 확인 필요 = 급여 미확정 → 급여 라우팅
- 직원별 지방소득세 내역 = 직원별 지급총액·소득세(국세)·지방소득세(특별징수)·상태 + 합계 행
- 신고지원과 동일한 값 안내(정합성)
- 책임 경계

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 준비 요약 | tenant·기간, 급여 집계 | 대상 인원·확인 필요·지방소득세 합계 | 없음(읽기) |
| 확인 필요 | 급여 미확정 | 사유 + CTA | 없음(읽기, 급여로 라우팅) |
| 직원별 지방소득세 내역 | `payrollEmployeeLine.localIncomeTaxKrw` 등 실제 기록값 | 직원별 지급총액·소득세(국세)·지방소득세(특별징수)·상태 | 없음(읽기 집계) |

지방소득세는 **read-only 집계 화면**이다(mutation 없음). v1은 원천세 특별징수분만이며, 종합소득세·법인세분 지방소득세·위택스 자동제출·신규 세액 계산은 범위 밖이다.

## 4k. 사업장현황신고 준비 화면 흐름 및 데이터 (JC-028)

진입: 면세 개인사업자에게만 노출되는 `연간신고 > 사업장현황신고` → 전용 화면.
연계: 사업자 유형(`tax_exempt`)과 자료수집(JC-009)·기장검토(JC-010)의 확정 거래 데이터를 묶어 수입금액, 매입/경비 자료, 누락/미확정 항목을 보여준다. 부가세 대상 사업자·법인은 해당 없음으로 처리한다.

핵심 경계: 이 화면은 **면세 개인사업자용 신고 준비 데이터**를 정리·검토하는 read-only 화면이다. 부가세 신고 대상자·법인·종합소득세/법인세분 신고는 범위 밖이며, 홈택스 제출·전자신고 파일 생성(JC-030)·자동제출(JC-023)은 하지 않는다.

구성(단일 스크롤·자료 중심):
- 상단 = 귀속연도 선택, 대상 여부·준비율·확인 필요·신고 준비 데이터 합계
- 확인 필요 = 자료 누락·계정분류 확인 → 자료수집/기장검토 라우팅
- 수입금액 분류 = 면세 매출/수입 계정별 집계 + 합계
- 매입·경비 자료 = 신고 참고 자료 준비 상태 + 누락 자료
- 홈택스 업로드·신고 전 확인 = 양식/파일에 들어갈 항목별 준비 상태와 직접 신고 handoff
- 책임 경계

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 대상 분기 | 사업자 유형(`tax_exempt`) | 면세 개인/해당 없음 상태 | 없음(읽기) |
| 준비 요약 | tenant·귀속연도, 자료수집/기장검토 집계 | 수입금액·매입/경비·미확정 거래·누락 자료 | 없음(읽기) |
| 확인 필요 | 자료 누락·계정분류 확인 필요 | 사유 + CTA | 없음(읽기, 자료수집/기장검토로 라우팅) |
| 수입금액 분류 | 확정 거래원장 | 계정/분류별 수입금액·상태 | 없음(읽기 집계) |
| 매입·경비 자료 | 수집 자료·확정 거래원장 | 자료 유형별 합계·누락 상태 | 없음(읽기 집계) |

사업장현황신고는 **read-only 준비 화면**이다(mutation 없음). v1은 면세 개인사업자만 대상으로 하며, 과세사업자·법인·전자신고 파일 생성·자동제출은 범위 밖이다.
