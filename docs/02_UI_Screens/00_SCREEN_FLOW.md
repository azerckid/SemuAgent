## 2. 전체 여정 (Company Home 중심)

```
로그인
  ├─▶ 신규 tenant 온보딩 완료
  │     └─ first-run sample dataset 자동 생성(샘플 표시 banner)
  └─▶ 회사 홈 (첫 진입 = 대시보드, 마케팅 페이지 아님)
        ├─ 현재 회계기간 상태 확인
        ├─ "다음 할 일"(신고 전 blockers)에서 액션 선택
        │     ├─▶ 기장검토 (미분류 거래)      [구현 완료]
        │     ├─▶ 자료수집 (미수집 자료)        [구현 완료]
        │     ├─▶ 부가세 (공제 검토)            [구현 완료]
        │     ├─▶ 급여 (확인 필요 직원)          [구현 완료]
        │     └─▶ 신고지원 (패키지·접수증)       [구현 완료]
        ├─ 준비 현황 카드 클릭 → 해당 워크스페이스로 이동
        └─ 최근 제출·영수증 → 전체 보기
```

## 3. 진입 · 전환 · 이탈

| 구분 | 내용 |
|:---|:---|
| 진입점 | 로그인 성공 직후 회사 홈으로 리다이렉트. 사이드바 "회사 홈" 클릭. |
| 핵심 CTA | ① 다음 할 일의 "…열기" 버튼 → 해당 워크스페이스. ② 준비 현황 카드 클릭 → 워크스페이스. ③ 상단 기간 선택 → 다른 회계기간으로 컨텍스트 전환. |
| 정상 이탈 | 워크스페이스로 이동하여 실제 작업 수행. |
| 이상 이탈 | 데이터 로드 실패 시 오류 상태 → "다시 시도". 자료 0건이면 빈 상태 → "첫 자료 업로드". |

## 4. 화면별 데이터 입출력 (회사 홈)

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 회계기간 상태 | 선택된 tenant·기간 | 진행률, 마감일, D-day | 없음(읽기) |
| 다음 할 일 | 미분류 거래 수, 미수집 자료, 공제 검토 수 | 액션 목록 + CTA | 없음(읽기, 클릭 시 라우팅) |
| 준비 현황 카드 | 워크스페이스별 집계 | 건수·상태칩 | 없음(읽기) |
| 최근 제출·영수증 | 제출/영수증 레코드 | 표(구분·항목·기간·상태·일시) | 없음(읽기) |
| 기간 선택 | 사용자 선택 | 활성 기간 | 세션/URL 컨텍스트 갱신 |

회사 홈은 **읽기 전용 대시보드**이다. 데이터 변경(mutation)은 각 워크스페이스에서 수행한다.

## 4b. 자료수집 화면 흐름 및 데이터

진입: 회사 홈 사이드바 "자료수집" 또는 "다음 할 일"의 "자료수집 열기" → 자료수집 화면.
복귀: 사이드바 "회사 홈" → 대시보드.

```
회사 홈 ──"자료수집 열기" / 사이드바──▶ 자료수집
   ▲                                      │
   └──────────── 사이드바 "회사 홈" ───────┘
```

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 수집 완결성 | tenant·기간, 자료 집계 | 진행률, 미수집 건수 | 없음(읽기) |
| 파일 업로드 | 사용자 파일(XLSX/CSV/PDF/이미지/ZIP) | 드롭존, 지원 형식 안내 | 업로드 → 파싱/정규화 큐 등록 |
| 자료유형 정규화 | 파싱 결과 | 유형별 건수·상태칩 | 정규화 확정/재분류 |
| 수집(가져오기) 상태 | 업로드 레코드 | 파일·유형·진행·상태·일시 | 재시도, 상세 보기 |
| 미수집·확인 필요 | 필수 자료 대비 부족분 | 항목 목록 + CTA | 재업로드/정규화 확인으로 라우팅 |

자료수집은 **업로드·정규화 mutation이 발생**하는 화면이다(회사 홈과 달리 읽기 전용 아님).
외부 고객 업로드 포털은 제외하고, 회사 내부 직접 업로드만 다룬다.

## 4c. 기장검토 화면 흐름 및 데이터

진입: 회사 홈 "다음 할 일"의 "기장검토 열기"(미검토 거래 분류) 또는 사이드바 "기장검토" → 기장검토 화면.
복귀: 사이드바 "회사 홈" 등으로 이동.

```
회사 홈 ──"기장검토 열기" / 사이드바──▶ 기장검토
   │                                        │ 거래 승인/수정
   │                                        ▼
   └──────────── 사이드바 ────────── 확정 전표(확정 탭)
```

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 분류 현황 | tenant·기간, 거래 집계 | 확정/대기 건수, 진행률 | 없음(읽기) |
| 분류 큐 | 정규화된 거래, AI 추천 계정과목·신뢰도 | 거래일·내용·상대처·금액·추천계정·신뢰도 | 계정과목 확정/변경 |
| 선택 거래 상세 | 선택 거래 | 분개 미리보기(차/대변), 기간 귀속, 증빙, 부가세 공제 | 없음(읽기, 승인 전 미리보기) |
| 승인 | 사용자 승인 액션 | 상태칩(검토 대기→확정) | 전표 확정(mutation), 확정 탭으로 이동 |
| 일괄 처리 | 다중 선택 | 선택 건수 | 일괄 승인/계정 변경(mutation) |

기장검토는 **분류 확정·승인 mutation이 발생**한다. AI 추천은 초안이며 확정 책임은 사용자에게 있다.
신뢰도 낮은 항목(예: 접대비/복리후생비 경계)은 "계정 지정"으로 강제 확인시킨다.

## 4d. 부가세 화면 흐름 및 데이터

진입: 회사 홈 "다음 할 일"의 "부가세 열기"(공제 검토) 또는 사이드바 "부가세" → 부가세 화면.
선행: 기장검토에서 거래가 확정되어야 매출·매입 집계가 정확해진다.

```
기장검토(확정 전표) ──집계──▶ 부가세
회사 홈 ──"부가세 열기" / 사이드바──▶ 부가세 ──검토 3건 완료──▶ 신고 패키지 생성
```

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 세액 요약 | 확정 전표 집계 | 매출세액 − 매입세액 = 납부(예정)세액, 마감 D-day | 없음(읽기) |
| 매출 구분 | 과세/영세율/면세 매출 | 그룹별 공급가액·매출세액 | 없음(읽기) |
| 매입세액 공제 검토 | 매입 거래, 불공제/안분 후보 | 공급가액·세액·공제 판정·사유 | 공제/불공제 확정, 공통매입 안분(mutation) |
| 부속 명세 | 합계표·수취명세서 등 | 서식별 준비 상태 | 없음(읽기, 패키지에 포함) |
| 신고 패키지 | 확정된 세액·명세 | 패키지 미리보기(PDF), 홈택스 입력 가이드 | 패키지 생성(mutation) |

부가세는 **공제 판정·안분·패키지 생성 mutation이 발생**한다. 불공제 후보 검토가 끝나야 최종 패키지를 생성할 수 있다.
**자동 홈택스 제출은 범위 밖**(신고 패키지 + 입력 가이드까지). 외부 세무사 검토 흐름은 v1 제외.

## 4e. 급여 화면 흐름 및 데이터

진입: 사이드바 "급여" → 급여 화면. 월별 급여 기간 선택.
연계: 원천징수 지급명세서 등은 **신고지원** 화면으로 전달된다.

```
급여 입력/자료 + 4대보험 고지내역 ──계산/매칭──▶ 급여대장 ──확인 필요 처리──▶ 급여 마감·확정
                                                        │ 지급명세서
                                                        ▼
                                                     신고지원
```

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 급여 요약 | 직원별 급여 입력·계산, 고지액 매칭 | 지급총액·공제총액·실지급액·마감상태 | 없음(읽기) |
| 확인 필요 알림 | 오류/누락 직원(취득일·보험료 고지 매칭 등) | 사유 + CTA | 해당 직원 편집으로 라우팅 |
| 급여대장 | 직원별 기본급·수당·공제 | 지급/원천세/4대보험/공제계/실지급 표 | 엑셀 내보내기, 개별 편집(mutation) |
| 공제 상세 | 원천세·4대보험 요율, 건강보험 EDI/사회보험 고지내역 | 항목별 공제 집계, 고지액 반영 여부 | 고지내역 업로드/수동 입력·매칭(mutation) |
| 명세서·마감 | 확정된 급여 | 급여명세서·지급명세서 미리보기, 마감 버튼 | 명세서 생성, 급여 마감·확정(mutation) |

급여는 **개별 편집·명세서 생성·마감 확정 mutation이 발생**한다. 확인 필요(오류/누락) 직원이 있으면 마감 전 처리해야 한다.
건강보험 EDI/사회보험 고지내역은 업로드 또는 수동 입력으로 반영하며, 자동 로그인·공동인증서 저장·자동 제출은 v1 제외다.
개인정보(급여·주민정보)를 다루므로 접근 권한과 마스킹은 구현 단계에서 확정한다. 외부 세무사 흐름은 v1 제외.

## 4f. 신고지원 화면 흐름 및 데이터

진입: 사이드바 "신고지원" 또는 각 신고 항목의 "…열기". 부가세·급여 화면에서 생성된 자료가 신고 항목으로 모인다.

```
부가세 패키지 ─┐
급여 지급명세서 ─┼─▶ 신고지원(항목·패키지) ─입력 가이드─▶ (회사가 홈택스 직접 제출) ─▶ 접수증 보관 ─▶ 사후 체크리스트
4대보험 자료 ──┘
```

책임 경계: **자동 제출·자동 납부 없음.** 첨부 패키지 생성 + 홈택스 단계별 입력 가이드 + 접수증 보관 + 사후 체크리스트까지.

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 신고 항목·패키지 | 부가세·급여·4대보험 산출물 | 항목별 상태(준비됨/대기/확인 필요)·패키지 | 패키지 생성(mutation), 선행 화면으로 라우팅 |
| 홈택스 입력 가이드 | 확정된 신고 값 | 단계별 입력 값(인원·지급액·세액) | 없음(읽기, 값 복사) |
| 제출 접수증 보관 | 사용자 업로드 접수증 | 접수증 목록·보관 상태 | 접수증 업로드·보관(mutation) |
| 사후 체크리스트 | 납부·보관 진행 | 체크 항목 완료 상태 | 체크 상태 갱신(mutation) |

신고지원은 **패키지 생성·접수증 보관·체크리스트 갱신 mutation이 발생**한다. 실제 제출/납부는 회사가 홈택스에서 직접 수행한다.

## 4h. First-run Sample Data 흐름 및 데이터

목적: 신규 사용자가 가입 직후 빈 화면 대신 승인 Preview와 유사한 샘플 업무 상태를 보고 메뉴 구조를 이해한다.

```
온보딩 완료 ──sample seed──▶ 회사 홈/각 워크스페이스(sample active)
       │                              │
       │                              └─▶ "샘플 데이터 삭제하고 실제 사용 시작"
       │                                             │
       └────────────── seed 실패 시 재시도 안내 ◀─────┘
```

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 샘플 생성 | 신규 tenant·사업장 | 승인 Preview와 유사한 채워진 상태 | sample dataset + domain row seed |
| 샘플 표시 | active sample dataset | 전역 banner, 샘플 badge, 삭제 CTA | 없음(읽기) |
| 샘플 삭제 | 사용자 확인 | 삭제 중/완료/실패 상태 | registry 기반 샘플 row 삭제 |
| 샘플 실패 | seed/delete 오류 | 재시도 CTA | 오류 사유 저장 |

샘플 데이터는 **학습용**이며 실제 신고 자료가 아니다. 삭제는 registry가 추적하는 샘플 행만 대상으로 하며, 실제 업로드·급여·신고 데이터는 삭제하지 않는다.

## 4g. 신고 준비 화면 흐름 및 데이터 (JC-029)

진입: 사이드바 "신고 준비"(허브) 또는 하위 **지급명세서·연말정산 / 지방소득세 / 사업장현황신고** 항목. 회사가 직접 신고하기 전에 홈택스·위택스에 넣을 확정 데이터가 준비됐는지 한 화면에서 확인한다.
연계: 공통 기반(자료수집·기장검토)에서 만들어진 확정 거래원장을 원천세·부가세·지급명세서/연말정산·지방소득세·사업장현황신고 병렬 트랙으로 넘기고, 각 트랙의 끝은 신고지원 또는 홈택스/위택스 직접 입력 handoff로 연결한다.

핵심 경계: 이 화면은 제출 버튼이 아니다. SemuAgent는 신고 데이터와 제출 보조 자료를 준비하고, 최종 신고·제출·납부는 사용자가 직접 수행한다. 자동제출은 JC-023 별도 결정 없이는 도입하지 않는다.

    자료수집 -> 기장검토 -> 확정 거래원장
                             ├─ 원천세(월)
                             ├─ 부가세(분기)
                             ├─ 지급명세서/연말정산(연)
                             ├─ 지방소득세(원천세 특별징수분)
                             └─ 사업장현황신고(면세 개인사업자)
                                          -> 신고지원(handoff)

구성:
- 상단 = 신고 데이터 준비율, 확인 필요 blocker, 다음 액션
- 중간 1 = 공통 기반(자료수집 -> 기장검토)의 입력·산출 상태
- 중간 2 = 병렬 신고 트랙(원천세·부가세·지급명세서/연말정산·지방소득세·사업장현황신고)의 입력·산출·handoff
- 하단 = 다가오는 세무 일정(D-day)과 직접 신고 책임 경계

| 트랙 | 입력 | 산출 | handoff |
|:---|:---|:---|:---|
| 자료수집 | 회사/담당자 업로드 | 수집 현황 + 누락 목록 | 기장 가능 상태 |
| 기장검토 | 수집자료 + 귀속기간 | 귀속월 확정·중복제거·계정분류 후보 | 확정 거래원장 |
| 원천세 | 급여대장·지급내역 | 간이세액표 집계 | 원천세 신고서 초안 수치 |
| 부가세 | 매출·매입 세금계산서 | 매입/매출세액 집계 + 검증 | 부가세 초안값 + 불일치 경고 |
| 지급명세서/연말정산 | 연간 지급내역·직원 명부 | 간이지급명세서 반기 집계·연말정산 준비 데이터·누락/검토 상태 | 신고 준비 데이터 확인 후 JC-030 파일 생성으로 연결 |
| 지방소득세 | 확정 본세·원천세 | 안분·세액 | 지방세 신고 수치 |
| 사업장현황신고 | 면세 수입금액·매입/경비 자료·사업자 유형 | 사업장현황신고 준비 요약·누락 자료·해당 없음 분기 | 홈택스 사업장현황신고 직접 입력 |

1차 구현 범위는 기존 트랙을 재프레임하고 메뉴/화면명을 바꾸는 것이다. 신규 산출 엔진, 신규 DB, 세액 계산, 자동제출은 이번 JC-029 범위 밖이다.

## 4i. 지급명세서·연말정산 준비 검토 화면 흐름 및 데이터 (JC-024)

진입: 신고 준비 허브(4g)의 "지급명세서/연말정산" 트랙 "열기" → 지급명세서·연말정산 화면(회사 홈 › 신고 준비 › 지급명세서·연말정산). 이 화면은 그 트랙을 roadmap→live로 채운다.
연계: 급여(JC-012)의 월별 원천·지급 데이터와 직원 명부(JC-015)를 반기/연 단위로 집계한다. 확인 필요(누락)는 급여·직원 명부로 라우팅한다.

핵심 경계: 이 화면은 **신고 준비 데이터**를 정리·검토하는 read-only 집계 화면이다. 정산액(결정세액·환급/추징) 계산·홈택스 제출은 범위 밖이다. **JC-030 전자신고 파일 생성 패널**(UI Preview)은 동일 화면에서 파일 후보·사전검증·다운로드 안내까지 제공하며, 홈택스 업로드·제출은 사용자가 직접 한다. 자동 제출·자격증명 저장은 하지 않는다.

구성(단일 스크롤·직원 중심):
- 상단 = 귀속연도/반기 선택, 대상 인원·확인 필요(누락)·데이터 준비 완료 수
- 확인 필요 = 급여 미확정·인적사항 누락 → 급여/직원 명부 라우팅
- 간이지급명세서(근로소득·반기) = 직원별 귀속기간·지급총액·원천징수세액·준비 상태
- **전자신고 파일 생성(JC-030)** = 데이터·규격 상태·4단계 흐름·사전검증·PII 1회 입력·홈택스 변환제출 안내
- 연말정산 준비·검토 = 직원별 재직·연간 지급합계·기납부 원천세·누락·검토 상태(정산액 계산은 범위 밖)
- 책임 경계

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 준비 요약 | tenant·기간, 급여/직원 집계 | 준비율·대상 인원·누락 수 | 없음(읽기) |
| 확인 필요 | 급여 미확정·인적사항 누락 | 사유 + CTA | 없음(읽기, 급여/직원 명부로 라우팅) |
| 간이지급명세서(근로·반기) | 월별 payroll 원천 집계 | 직원별 지급총액·원천징수세액·상태 | 없음(읽기 집계) |
| 연말정산 준비·검토 | 직원별 연간 지급·기납부 | 연간 합계·기납부 원천세·누락·검토상태 | 없음(읽기 집계) |

지급명세서·연말정산은 **read-only 집계 화면**이다(mutation 없음). v1은 간이지급명세서(근로소득)·연말정산 데이터·검토까지이며, 정산액 계산·전자신고 파일 생성·자동제출은 범위 밖이다.

## 4j. 지방소득세 준비 화면 흐름 및 데이터 (JC-027)

진입: 신고 준비 허브(4g)의 "지방소득세" 트랙 "검토 화면" → 지방소득세 화면(회사 홈 › 신고 준비 › 지방소득세). 허브의 마지막 roadmap 트랙을 live로 채운다.
연계: 급여(JC-012)에 이미 기록된 `payrollEmployeeLine.localIncomeTaxKrw`(원천세 특별징수분)를 원천세 신고 주기 기준으로 집계한다. 확인 필요(급여 미확정)는 급여로 라우팅한다.

핵심 경계: 이 화면은 **원천세 특별징수분 지방소득세만** 다룬다(종합소득세분·법인세분은 범위 밖, JC-025/026 이후). 실제 기록된 값을 집계할 뿐 **신규 세액 계산이 아니다**(근사 계산 아님). 신고지원(JC-013)이 표시하던 근사치(`splitWithholdingTax`)를 이 화면과 같은 실제값으로 교체해, 두 화면이 같은 기간에 다른 숫자를 보이지 않게 한다. 위택스 제출은 사용자가 직접 수행하며 자동 제출·자격증명 저장은 하지 않는다.

구성(단일 스크롤·직원 중심):
- 상단 = 귀속기간 선택(원천세 신고 주기 기준), 대상 인원·확인 필요·지방소득세 합계
- 확인 필요 = 급여 미확정 → 급여 라우팅
- 직원별 지방소득세 내역 = 직원별 지급총액·소득세(국세)·지방소득세(특별징수)·상태 + 합계 행
- 신고지원과 동일한 값 안내(정합성)
- 책임 경계

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 준비 요약 | tenant·기간, 급여 집계 | 대상 인원·확인 필요·지방소득세 합계 | 없음(읽기) |
| 확인 필요 | 급여 미확정 | 사유 + CTA | 없음(읽기, 급여로 라우팅) |
| 직원별 지방소득세 내역 | `payrollEmployeeLine.localIncomeTaxKrw` 등 실제 기록값 | 직원별 지급총액·소득세(국세)·지방소득세(특별징수)·상태 | 없음(읽기 집계) |

지방소득세는 **read-only 집계 화면**이다(mutation 없음). v1은 원천세 특별징수분만이며, 종합소득세·법인세분 지방소득세·위택스 자동제출·신규 세액 계산은 범위 밖이다.

## 4k. 사업장현황신고 준비 화면 흐름 및 데이터 (JC-028)

진입: 신고 준비 허브(4g)의 "사업장현황신고" 트랙 "검토 화면" → 사업장현황신고 화면(회사 홈 › 신고 준비 › 사업장현황신고). 면세 개인사업자가 부가세 신고 대신 홈택스 사업장현황신고에 넣을 수입금액·자료 상태를 확인하는 전용 화면이다.
연계: 사업자 유형(`tax_exempt`)과 자료수집(JC-009)·기장검토(JC-010)의 확정 거래 데이터를 묶어 수입금액, 매입/경비 자료, 누락/미확정 항목을 보여준다. 부가세 대상 사업자·법인은 해당 없음으로 처리한다.

핵심 경계: 이 화면은 **면세 개인사업자용 신고 준비 데이터**를 정리·검토하는 read-only 화면이다. 부가세 신고 대상자·법인·종합소득세/법인세분 신고는 범위 밖이며, 홈택스 제출·전자신고 파일 생성(JC-030)·자동제출(JC-023)은 하지 않는다.

구성(단일 스크롤·자료 중심):
- 상단 = 귀속연도 선택, 대상 여부·준비율·확인 필요·신고 준비 데이터 합계
- 확인 필요 = 자료 누락·계정분류 확인 → 자료수집/기장검토 라우팅
- 수입금액 분류 = 면세 매출/수입 계정별 집계 + 합계
- 매입·경비 자료 = 신고 참고 자료 준비 상태 + 누락 자료
- 홈택스 입력 전 확인 = 입력 항목별 준비 상태와 직접 신고 handoff
- 책임 경계

| 항목 | 입력 | 표시 | 저장/전송 |
|:---|:---|:---|:---|
| 대상 분기 | 사업자 유형(`tax_exempt`) | 면세 개인/해당 없음 상태 | 없음(읽기) |
| 준비 요약 | tenant·귀속연도, 자료수집/기장검토 집계 | 수입금액·매입/경비·미확정 거래·누락 자료 | 없음(읽기) |
| 확인 필요 | 자료 누락·계정분류 확인 필요 | 사유 + CTA | 없음(읽기, 자료수집/기장검토로 라우팅) |
| 수입금액 분류 | 확정 거래원장 | 계정/분류별 수입금액·상태 | 없음(읽기 집계) |
| 매입·경비 자료 | 수집 자료·확정 거래원장 | 자료 유형별 합계·누락 상태 | 없음(읽기 집계) |

사업장현황신고는 **read-only 준비 화면**이다(mutation 없음). v1은 면세 개인사업자만 대상으로 하며, 과세사업자·법인·전자신고 파일 생성·자동제출은 범위 밖이다.
